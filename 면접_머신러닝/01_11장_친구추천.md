# 11장. 친구 추천 (Friend Recommendation)

## 핵심 개념 요약

친구 추천 시스템은 소셜 네트워크에서 **아직 연결되지 않은 두 사용자 사이에 친구 관계가 형성될 가능성**을 예측하는 문제이다. 이는 그래프 이론에서 **Link Prediction** 문제에 해당하며, GNN(Graph Neural Network)을 활용하면 그래프 구조와 노드 특징을 동시에 학습할 수 있다.

### 전체 파이프라인

```
[소셜 그래프 구성] → [노드 특징 추출] → [GNN 임베딩 학습] → [Edge 점수 계산] → [친구 추천 리스트 생성]
```

## Q&A

### Q1. GNN은 이진 분류 모델이라고 판단해도 되는가?

결론부터 말하면, **"GNN은 이진 분류 모델 그 자체라기보다, 그래프 데이터를 학습하여 다양한 작업(Task)을 수행할 수 있는 범용적인 신경망 아키텍처"**라고 이해하는 것이 더 정확하다.

다만, 친구 추천(Link Prediction) 관점에서만 본다면 **결과적으로 이진 분류(Binary Classification)의 형태**를 띠게 된다.

#### 1) GNN은 '임베딩 생성기'에 가깝다

GNN의 핵심 역할은 그래프의 구조와 노드의 속성을 결합하여 각 노드를 수치화된 벡터(Embedding)로 만드는 것이다.

- **입력:** 그래프 구조(인접 행렬) + 노드 특징(Feature)
- **과정:** 이웃 노드들의 정보를 모으고(Aggregate) 내 정보와 합치는(Update) 과정을 반복
- **출력:** 각 노드의 특징이 응축된 **저차원 벡터(Embedding)**

#### 2) GNN이 수행하는 3가지 주요 작업

GNN은 생성된 임베딩을 '어떤 레이어'에 연결하느냐에 따라 다른 모델이 된다.

| 작업 수준 (Level) | 설명 | 예시 | 분류 형태 |
|---|---|---|---|
| **Node Level** | 각 노드의 정체성을 예측 | 스팸 사용자 판별, 사용자 관심사 분류 | 이진/다중 분류 |
| **Edge Level** | 두 노드 사이의 연결 가능성 예측 | 친구 추천 (Link Prediction) | 이진 분류 |
| **Graph Level** | 그래프 전체의 특성을 예측 | 화합물의 독성 여부, 커뮤니티 성격 판단 | 이진/다중 분류 |

#### 3) 왜 친구 추천에서는 '이진 분류'처럼 보일까?

1. **임베딩 추출:** GNN을 통해 사용자 A와 사용자 B의 벡터($h_A, h_B$)를 얻는다.
2. **점수 계산:** 두 벡터를 내적(Inner Product)하거나 MLP(다층 퍼셉트론)에 넣는다.
   - $Score = \text{MLP}(h_A, h_B)$ 또는 $Score = h_A \cdot h_B$
3. **최종 판단:** 이 점수를 시그모이드(Sigmoid) 함수에 통과시켜 0과 1 사이의 확률값으로 만든다.
   - **1에 가까우면:** 친구가 될 확률 높음 (Positive)
   - **0에 가까우면:** 친구가 될 확률 낮음 (Negative)

즉, **GNN은 "특징 추출기(Feature Extractor)"** 역할을 하고, 그 뒤에 붙는 **최종 레이어가 "이진 분류기"** 역할을 수행하는 구조이다.

#### 4) 수학적 이해 (간략화)

GNN의 한 레이어에서 정보가 업데이트되는 과정:

$$h_v^{(k)} = \sigma \left( W \cdot \text{MEAN} \left( \{h_v^{(k-1)}\} \cup \{h_u^{(k-1)}, \forall u \in N(v)\} \right) \right)$$

여기서 생성된 $h_v$를 가지고 최종적으로 이진 분류를 할지, 회귀(Regression)를 할지는 설계자의 몫이다.

#### 요약

- **GNN 자체**는 이미지의 CNN이나 텍스트의 Transformer처럼 **데이터의 형태(그래프)에 최적화된 아키텍처**이다.
- 하지만 **친구 추천 시스템**에서는 GNN을 활용해 **"연결됨(1) vs 연결 안 됨(0)"을 예측하는 이진 분류 모델**로 구성하여 사용하는 것이 일반적이다.

### Q2. GNN 이전에는 친구 추천을 어떻게 했는가?

GNN 이전의 전통적인 방법들:

| 방법 | 설명 | 한계 |
|---|---|---|
| **Common Neighbors** | 공통 친구 수가 많을수록 추천 | 구조 정보만 사용, 노드 특징 무시 |
| **Jaccard Coefficient** | 공통 이웃 / 전체 이웃 비율 | 단순 비율이라 네트워크 규모 차이에 약함 |
| **Adamic-Adar Index** | 공통 이웃의 연결 수에 반비례하는 가중치 부여 | 2-hop까지만 고려 |
| **Preferential Attachment** | 연결이 많은 노드가 더 많은 연결을 가짐 (부익부) | 인기 편향 존재 |
| **Matrix Factorization** | 인접 행렬을 저차원으로 분해 | 비선형 패턴 포착 어려움 |
| **Node2Vec / DeepWalk** | 랜덤 워크 기반 노드 임베딩 | 그래프 구조만 활용, 노드 특징 활용 불가 |

**GNN의 장점:** 그래프 구조 + 노드 특징(프로필, 활동 내역 등)을 **동시에** 학습할 수 있다.

### Q3. GNN의 대표적인 변형 모델들의 차이점은?

| 모델 | Aggregation 방식 | 특징 |
|---|---|---|
| **GCN** (Graph Convolutional Network) | 이웃 노드 평균 (Mean) | 가장 기본적, spectral 기반에서 출발 |
| **GraphSAGE** | Sample & Aggregate (Mean/LSTM/Pool) | 대규모 그래프에 적합, inductive learning 가능 |
| **GAT** (Graph Attention Network) | Attention 가중 합산 | 이웃마다 다른 중요도 부여 가능 |
| **GIN** (Graph Isomorphism Network) | Sum + MLP | 이론적으로 가장 강력한 표현력 (WL test와 동급) |

#### 면접 포인트
- **GCN vs GAT:** "모든 이웃을 동등하게 볼 것인가(GCN) vs 중요한 이웃에 더 집중할 것인가(GAT)"
- **GraphSAGE:** 전체 그래프를 메모리에 올리지 않고 **샘플링**으로 학습 → 수억 노드의 실제 서비스에 적합
- **Inductive vs Transductive:** GraphSAGE는 학습 시 보지 못한 새로운 노드에도 임베딩 생성 가능 (inductive)

### Q4. Message Passing이란 무엇인가?

GNN의 핵심 동작 원리를 **Message Passing Neural Network (MPNN)** 프레임워크로 통합하여 설명할 수 있다.

```
각 레이어에서:
1. Message:   m_v = AGGREGATE({h_u : u ∈ N(v)})    ← 이웃 정보 수집
2. Update:    h_v' = UPDATE(h_v, m_v)               ← 자신의 정보와 결합
3. Readout:   (필요 시) 그래프 전체 표현 생성
```

- **k번의 레이어** = k-hop 이웃까지의 정보를 반영
- 레이어가 많을수록 멀리 있는 노드 정보를 가져오지만, **Over-smoothing** 문제 발생 가능

#### Over-smoothing 문제
- 레이어를 너무 깊게 쌓으면 **모든 노드의 임베딩이 비슷해지는 현상**
- 해결 방법: Residual Connection, JumpingKnowledge, DropEdge 등

### Q5. 친구 추천 시스템의 실무 설계 시 고려사항은?

#### 1) 데이터 불균형 문제
- 실제 소셜 네트워크에서 연결된 엣지(Positive)보다 연결되지 않은 엣지(Negative)가 **압도적으로 많다**
- **해결:** Negative Sampling 전략 활용 (랜덤 샘플링, Hard Negative Mining 등)

#### 2) 확장성 (Scalability)
- 수억 명의 유저 그래프를 한 번에 학습하기 어렵다
- **해결:** Mini-batch 학습 (GraphSAGE의 Neighbor Sampling), Cluster-GCN, GraphSAINT 등

#### 3) Cold Start 문제
- 신규 유저는 연결(엣지)이 없어 그래프 기반 추천이 어렵다
- **해결:** 프로필 정보(나이, 위치, 관심사 등)를 노드 특징으로 활용, 콘텐츠 기반 추천과 하이브리드

#### 4) 실시간 서빙
- 학습된 모델로 실시간 추천을 제공해야 한다
- **해결:** 임베딩을 사전 계산하여 저장(ANN 인덱스: Faiss, ScaNN 등) → 유사도 기반 빠른 검색

#### 5) 평가 지표
| 지표 | 설명 |
|---|---|
| **AUC-ROC** | 이진 분류 성능의 전반적 평가 |
| **Hit Rate@K** | 상위 K개 추천 중 실제 친구가 포함된 비율 |
| **MRR** (Mean Reciprocal Rank) | 정답이 추천 리스트에서 몇 번째에 나오는지 |
| **Precision@K / Recall@K** | 추천 정밀도와 재현율 |

### Q6. Link Prediction의 손실 함수(Loss Function)는?

#### Binary Cross-Entropy Loss (가장 일반적)
$$\mathcal{L} = -\frac{1}{N}\sum_{i=1}^{N} \left[ y_i \log(\hat{y}_i) + (1-y_i) \log(1-\hat{y}_i) \right]$$

- $y_i$: 실제 레이블 (1=연결됨, 0=연결 안 됨)
- $\hat{y}_i$: 모델이 예측한 연결 확률

#### BPR Loss (Bayesian Personalized Ranking)
$$\mathcal{L} = -\sum_{(u,i,j)} \log \sigma(\hat{y}_{ui} - \hat{y}_{uj})$$

- Positive 엣지의 점수가 Negative 엣지보다 높도록 학습
- 추천 시스템에서 **랭킹 기반** 학습 시 효과적

#### Contrastive Loss
- Positive pair는 가깝게, Negative pair는 멀게 임베딩 학습
- Self-supervised learning과 결합 시 많이 사용

## 면접 대비 한줄 정리

| 키워드 | 한줄 정리 |
|---|---|
| GNN | 그래프 구조 데이터를 학습하는 범용 신경망 아키텍처 |
| Link Prediction | 두 노드 사이 엣지 존재 여부를 예측하는 문제 |
| Message Passing | 이웃 노드의 정보를 수집하고 자신의 표현을 업데이트하는 GNN의 핵심 메커니즘 |
| Over-smoothing | GNN 레이어가 깊어지면 노드 표현이 획일화되는 현상 |
| Negative Sampling | 불균형 데이터에서 부정 샘플을 전략적으로 선택하는 기법 |
| Inductive Learning | 학습 시 보지 못한 새 노드에도 일반화 가능한 학습 방식 |
| GraphSAGE | 샘플링 기반으로 대규모 그래프에 확장 가능한 GNN |
| GAT | 어텐션으로 이웃의 중요도를 차별화하는 GNN |
