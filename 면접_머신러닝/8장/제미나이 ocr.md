# 제8장 소셜 플랫폼 광고 클릭 예측

## 1. 소개
온라인 광고를 통해 광고주는 노출수, 클릭 수, 전환 수 등 측정 가능한 반응을 얻기 위해 플랫폼에 입찰하고 광고를 게재한다.  
구글, 메타, 인스타그램과 같이 많은 온라인 플랫폼은 기본적으로 사용자에게 관련성 높은 광고를 표시한다.  
이 장에서는 소셜 미디어 플랫폼과 유사한 광고 클릭 예측(CTR 예측) 시스템을 설계한다.

---

## 2. 요구사항 명확화

- **비즈니스 목표**: 수익 극대화  
- **광고 환경**: 사용자의 타임라인에만 게재되며 클릭 시 일정한 수익이 발생한다  
- **사용자 상호작용**: 동일 광고의 반복 노출을 허용하며, 피로 기간(fatigue period)은 없다고 가정한다  
- **부정적 피드백**: '이 광고 숨기기' 기능을 지원하여 관련 없는 광고를 감지한다  
- **지속적 학습**: 모델 업데이트가 5분만 지연되어도 성능이 저하될 수 있으므로 지속적인 학습이 필수적이다

---

## 3. 머신러닝 문제 구조화

### 목표 정의
사용자가 클릭할 가능성이 높은 광고를 표시하여 수익을 높이는 머신러닝 목표로 전환한다.

### 입출력 지정
- **입력**: 사용자 정보를 입력으로 받는다  
- **출력**: 클릭 확률에 따라 순위가 매겨진 광고 목록을 출력한다

### 머신러닝 유형 선택
- **Pointwise LTR (Learning to Rank)**  
  `<사용자, 광고>` 쌍을 입력으로 받아 사용자가 광고를 클릭할지 여부를 예측하는 이진 분류 모델을 사용한다.

---

## 4. 데이터 준비

### 원시 데이터

- **광고 데이터**  
  광고 ID, 광고주 ID, 캠페인 ID, 범주, 이미지/동영상 URL 등  

- **사용자 데이터**  
  ID, 연령, 성별, 도시, 국가, 언어, 타임존 등  

- **상호작용 데이터**  
  노출, 클릭, 전환, 체류 시간, 위치, 타임스탬프  

### 피처 엔지니어링

- **광고 피처**
  - ID 임베딩 계층 사용  
  - 사전 학습 모델(SimCLR 등)을 통한 이미지/동영상 벡터화  
  - 범주 정보  
  - 노출/클릭 수 통계  

- **사용자 피처**
  - 인구통계학적 정보  
  - 상황별 정보(단말기, 시간)  
  - 과거 상호작용 통계(클릭한 광고 리스트, 클릭률 등)

---

## 5. 모델 개발

### 모델 선택 후보
로지스틱 회귀(LR), 피처 크로싱 + LR, GBDT, GBDT + LR, 신경망(NN), DCN, FM, DeepFM 등이 있다.

### Factorization Machines (FM) 공식

$$
\hat{y}(x)=w_{0}+\sum_{i}w_{i}x_{i}+\sum_{i}\sum_{j}\langle v_{i},v_{j}\rangle x_{i}x_{j}
$$

- 처음 두 항은 피처의 선형 조합을 계산하고  
- 세 번째 항은 쌍별 피처 상호작용을 모델링한다

### DeepFM
DNN(고차 피처 포착)과 FM(저수준 쌍별 피처 상호작용 포착)을 결합하여 성능을 극대화한다.

---

## 6. 평가 및 서빙

### 오프라인 지표

- **크로스 엔트로피 (CE)**  
  예측 확률이 실제 라벨에 얼마나 가까운지 측정한다  

- **NCE (Normalized Cross-Entropy)**  
  모델의 CE와 단순 기준선(평균 클릭률)의 CE 사이의 비율이다  

### 서빙 아키텍처

- **데이터 준비 파이프라인**  
  배치 피처(정적)와 온라인 피처(동적)를 계산하여 저장한다  

- **지속적 학습 파이프라인**  
  새로운 데이터를 통해 모델을 미세 조정(Fine-tuning)한다  

- **예측 파이프라인**  
  후보 생성(필터링) → 랭킹(순위 매기기) → 재순위화(로직 보정)

---

## 7. 추가 논의 주제

- 데이터 누수(Data Leakage) 방지  
- 모델 보정(Calibration) 기법  
- 지속적 학습 시 발생하는 파괴적 망각(Catastrophic Forgetting) 문제
